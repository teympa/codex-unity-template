# 貢献ガイド

この repo は、Unity 開発を Codex で進めるためのスターターテンプレートです。

目的は「小さく、実用的で、初心者でも導入しやすいこと」です。大きな仕組みを増やすより、分かりやすい改善を優先してください。

## 役立つ改善例

- セットアップ説明の改善
- 初心者向け導線の改善
- Unity 向けルールの明確化
- validator のノイズを増やさない改善
- Codex 依頼例の追加

## 変更前の考え方

1. 多くの Unity プロジェクトで再利用できるか
2. 特定プロジェクト専用の前提になっていないか
3. hooks と CI が重くなりすぎないか
4. 強すぎる自動化ではなく、軽い柵になっているか

## 変更ルール

- `AGENTS.md` は短く実用的に保つ
- 繰り返しレビュー手順は `skills/` に置く
- 機械的なチェックは hooks か CI に置く
- 必要性が明確でない依存追加は避ける
- 大きいリファクタと無関係な docs 修正を混ぜすぎない

## おすすめの変更フロー

1. 構造に関わるなら先に design-review 的に考える
2. 小さい単位で変更する
3. 次を実行する

```bash
python scripts/validate_unity_gameplay.py
```

4. docs を変えたなら初心者目線で一度読み返す
5. pull request では次を短く書く
   - 何を変えたか
   - なぜ必要か
   - 利用者が調整すべき点があるか

## pull request チェック

- 再利用しやすさを保てている
- 初心者に伝わる説明になっている
- validator と hooks が軽いまま
- パスや例が想定構成と合っている
- 新しいルールの意図が説明されている
