# Codex 依頼例

このファイルでは、このテンプレートを使う Unity プロジェクトで Codex にどう頼むと進めやすいかの例をまとめます。

## 1. 小さな gameplay 変更を頼む

```text
プレイヤーにダッシュのクールダウンを追加してください。
Assets/Scripts/Gameplay と Assets/Scripts/UI だけ触ってください。
バランス値はコードへ直書きしないでください。
UI は表示と入力の橋渡しだけにしてください。
```

## 2. 実装前に design-review を頼む

```text
design-review を使ってこの仕様を見てください。
プレイヤーは攻撃ボタン長押しで溜め攻撃を行います。
早く離すと小ダメージ、最大まで溜めると大ダメージと反動があります。
state の持ち方、UI との境界、調整値の置き場を見てほしいです。
```

## 3. 実装後に code-review を頼む

```text
変更した gameplay ファイルに対して code-review をしてください。
責務分離、毎フレーム処理のコスト、直書き値、
state 遷移、Unity 参照の取り方の危険さを中心に見てください。
```

## 4. 安全なリファクタを頼む

```text
EnemyController を薄くする方向でリファクタしてください。
可能なら純粋ロジックを通常クラスへ分離してください。
挙動変更は必要最小限にしてください。
編集前に目的、対象ファイル、リスクを短くまとめてください。
```

## 5. UI 追加を頼む

```text
スタミナバー UI を追加してください。
UI は gameplay state の変化を受けて表示更新するだけにしてください。
gameplay ルールを UI に書かないでください。
```

## コツ

- 触る範囲を明示する
- 挙動変更の可否を書く
- 実装前レビューが欲しいか書く
- 実装後レビューが欲しいか書く
